# Proof Freshness Fix Plan

## Problem Analysis
The proof freshness check is failing with a tree hash mismatch:
- Expected hash (from release_gate.json): f2ca81ad4edb2935b15c168f09d1184d3009575461f4107a6d4bf984103c20d6
- Actual hash (current tree): 97e57b091f20eacd8cb26bd51762f873c7c4ec1612e2623de01ce10e57d0ecb3

This indicates that the proof artifacts were generated from a different repository state than what is currently checked out.

## Root Cause
The proof_manifest.json shows it was generated at "2026-06-02T06:41:14.899507+00:00", but the current tree has changed since then. Even though `git status --porcelain` shows no changes, there may be:
1. Untracked files affecting the proof input
2. The proof was generated from a different commit/branch
3. Clock/filesystem issues causing hash inconsistencies

## Solution
Regenerate all proof artifacts from the current tree state using the established proof generation process.

## Implementation Steps
1. Run the full proof generation pipeline: `make proof`
   - This will run test-backend-proof, test-frontend-proof, release-proof-local, and all verification steps
   - It will update artifacts/proof/current/ with fresh proofs based on current tree
2. Verify proof freshness passes: `python3 scripts/check_proof_freshness.py`
3. Verify proof consistency: `python3 scripts/check_proof_consistency.py`
4. Verify required logs are present: `python3 scripts/check_required_proof_logs.py --strict-required-files`
5. Verify release handoff consistency: `python3 scripts/check_release_handoff_consistency.py --archive dist/JUDGE_ATLAS-main-final.zip`

## Expected Outcome
After running `make proof`:
- proof_input_tree_hash in release_gate.json should match the current tree hash
- check_proof_freshness.py should return PASS
- All related proof validation checks should pass
- The release artifact (dist/JUDGE_ATLAS-main-final.zip) will contain verifiable proofs
