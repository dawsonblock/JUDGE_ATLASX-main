# REPO_BOUNDARY

This document defines what is and is not part of the JUDGE_ATLASX runtime product.

## Runtime product scope

The following top-level directories and files are part of the release archive:

| Path | Purpose |
|---|---|
| `backend/` | Python FastAPI backend, Alembic migrations, source registry |
| `frontend/` | Next.js 14 frontend |
| `scripts/` | CI/release/proof tooling |
| `docs/` | Product documentation and proof summaries |
| `infra/` | Infrastructure configuration |
| `demo/` | Synthetic demonstration environment |
| `.github/` | CI workflows |
| `artifacts/proof/current/` | Approved proof summaries (generated, not authored) |
| `README.md`, `Makefile`, `docker-compose.yml`, etc. | Root config files |

## Non-runtime scope (NEVER shipped)

The following are NOT part of the runtime product and MUST NOT appear in any release archive:

| Path | Reason |
|---|---|
| `research/` | Third-party research repos — read-only reference, never runtime |
| `Research /` | Trailing-space variant of research/ — hard-blocked |
| `Research/` | Case variant of research/ — hard-blocked |
| `external/` | External vendored dependencies — reference only |
| `external_reference/` | External reference material |
| `node_modules/` | npm build artifact |
| `frontend/.next/` | Next.js build output |
| `frontend/dist/` | Frontend build output |
| `dist/` | Release build output folder |
| `build/` | Build output |
| `.venv/`, `backend/.venv/` | Python virtual environments |
| `__pycache__/`, `*.pyc` | Python bytecode |
| `.pytest_cache/`, `.mypy_cache/`, `.ruff_cache/` | Tool caches |
| `logs/`, `tmp/`, `temp/` | Ephemeral runtime data |
| `evidence_store/`, `data/local/` | Local data stores |
| `.env`, `.env.*` | Secrets and local config (only `.env.example` ships) |
| `*.pem`, `*.key`, `*.p12` | Private keys |
| `*.db`, `*.sqlite`, `*.sqlite3` | Database files |
| `artifacts/proof/archive/` | Historical proof archive |
| `archive_validation.md`, `archive_validation.log` | Self-referential validation outputs |

## Research / reference policy

Research and reference material exists to inform the product, not to ship with it.

- Useful ideas from research must be implemented as normal commits to `backend/`, `frontend/`, or `scripts/`.
- Research repos must not be copied into runtime paths.
- Research repos must not contain secrets, credentials, or production data.
- The `research/` tree is excluded from:
  - Release archive validation
  - Generated-file guard
  - Truth-claim scanner
  - Path hygiene scan (but violations inside it are noted separately)

## Release archive rules

1. The ONLY valid release artifact is one generated by `scripts/build_release_archive.py`.
2. Do not manually ZIP the workspace folder.
3. The clean archive must pass `scripts/validate_release_archive.py` with `valid: true, errors: 0`.
4. The validator rejects any archive member with a trailing-whitespace path component.
5. The validator rejects any embedded absolute local developer paths (for example `<user-home>/...` or `<windows-user-home>\\...`).

## Acceptance criteria for a releaseable archive

- File count: ~700–1200 files
- Compressed size: ~1–10 MB
- Validation result: PASS
- No `research/`, `Research /`, or `external/` inside the archive
- No embedded local developer machine paths
- No `.env`, `.pem`, `.key`, `.db`, or `.log` files
- Contains `artifacts/proof/current/CURRENT_PROOF.md` and `release_gate.json`

## Enforcement

These boundaries are enforced by:

| Script | What it checks |
|---|---|
| `scripts/build_release_archive.py` | Builds only runtime files using allowlist |
| `scripts/validate_release_archive.py` | Rejects non-runtime paths, secrets, local paths |
| `scripts/check_path_hygiene.py` | Rejects trailing-whitespace path components |
| `scripts/check_no_generated_files.py` | Rejects committed generated/cache files |
| `.gitignore` | Prevents tracking of non-runtime trees |
| `.dockerignore` | Prevents non-runtime files from entering Docker images |
| `.github/workflows/quality-gate.yml` | Runs all guards on every push/PR |
