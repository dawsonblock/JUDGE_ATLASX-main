## Phase 1 — Fix Node Policy: Flexible Local, Pinned CI/Container

**Problem:** `frontend/package.json` uses exact `"node": "22.22.3"` which fails `check_node_policy.py` on any valid Node 22.x patch (e.g., 22.16.0). This is brittle for no real benefit.

**Fix:**
| File | Current | New |
|---|---|---|
| `frontend/package.json` engines.node | `"22.22.3"` | `">=22 <23"` |
| `.nvmrc` | `22.22.3` | `22` |
| `frontend/.nvmrc` | `22.22.3` | `22` |
| `frontend/Dockerfile` | `FROM node:22.22.3-bookworm-slim` | **keep pinned** |
| `.github/workflows/*` node-version | `"22"` | **keep as-is** |
| `scripts/check_frontend_node_gate.py` default | `22` | **keep as-is** |

This allows local dev on any Node 22.x while keeping Docker CI pinned for reproducibility.

## Phase 3 — Regenerate Proof Artifacts from Current Tree

**Problem:** 53 of 54 referenced proof logs are missing. The archive contains proof JSON without the referenced logs.

**Steps:**
1. Delete stale proof output: `rm -rf artifacts/proof/current && mkdir -p artifacts/proof/current`
2. Install Node 22 via nvm: `nvm install 22 && nvm use 22`
3. Ensure Python 3.11 is active: `pyenv shell 3.11.9` or `uv python install 3.11 && uv venv`
4. Run backend tests under Python 3.11: `python3 -m pytest backend/tests`
5. Run frontend clean install: `cd frontend && rm -rf node_modules .next && npm ci`
6. Run frontend gates: `npm run lint`, `npm run typecheck`, `npm run test:contracts`, `npm run build`
7. Run Docker smoke test (or honestly mark unavailable): `docker compose up -d --build && curl -f http://localhost:3000 && curl -f http://localhost:8000/health && docker compose down`
8. Run canonical proof: `make proof` or `bash scripts/package_and_validate_release_archive.sh`
