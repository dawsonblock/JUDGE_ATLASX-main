## Priority 9: Separate BLOCKED from FAILED Gates

**Problem:** Docker/PostGIS unavailable recorded same as failing test.

**Fix:** Use clean statuses:
- `PASS` — gate ran and succeeded
- `FAIL` — gate ran and failed (code broken)
- `BLOCKED` — gate could not run (environment missing)
- `SKIPPED_WITH_REASON` — gate skipped with documented reason

**Example:**
```json
{
  "gate": "docker_smoke",
  "status": "blocked",
  "reason": "Docker daemon unavailable"
}
```
