# Release Artifact Repair Plan — Fix Proof Packaging

## Problem Analysis

The uploaded ZIP (`JUDGE_ATLASX-main-master 2.zip`) is a manual source snapshot, not a canonical release artifact. Key blockers:

1. **Missing proof logs in archive** — `build_release_archive.py` excludes `.log` files via `EXCLUDED_SUFFIXES` even when they're required proof artifacts
2. **Wrong archive root name** — Expected `JUDGE_ATLAS-main`, got `JUDGE_ATLASX-main-master`
3. **Non-canonical archive** — Missing `dist/JUDGE_ATLAS-main-final.zip`
4. **Node 20 doc inconsistencies** — Some docs still reference Node 20 while code requires Node 22

## Root Cause

In `scripts/build_release_archive.py`:
- Line 96-108: `.log` files are in `EXCLUDED_SUFFIXES` (line 102)
- The `_collect_files` function applies `_is_excluded()` check AFTER receiving `packaged_proof_paths`
- This causes proof logs to be excluded even when explicitly required

## Repair Steps

### Step 1: Fix Archive Build Script (Critical)

**File: `scripts/build_release_archive.py`**

The fix: Add logic to NOT exclude `.log` files that are in the packaged proof paths set. The exclusion check on line 221 needs to respect the `packaged_proof_paths` parameter.

```python
# In _is_excluded(), add before the suffix check (around line 221):
if rel_path in packaged_proof_paths or any(
    rel_path == p or rel_path.endswith("/" + p.split("/")[-1])
    for p in packaged_proof_paths
):
    # This is a required proof log, do not exclude based on suffix alone
    return False
```

Or alternatively: Remove `.log` from `EXCLUDED_SUFFIXES` and rely on the explicit exclusion lists instead.

**Decision point:** Remove `.log` from `EXCLUDED_SUFFIXES` since proof logs are explicitly managed through `packaged_proof_paths`.
